# VFA Front-End Website Constitution

## Core Principles

### I. Component-Driven Development
All UI features must be implemented as reusable, self-contained components. Components should be independently testable and documented.

### II. API-First Data Access
All dynamic data must be fetched from backend APIs. No direct database or server logic in the front-end codebase.

### III. Test-First
Automated tests must be written for all components and features. Tests must be run and pass before deployment.

### IV. Accessibility & Responsiveness
All UI must be accessible (WCAG 2.1 AA) and responsive across devices.

### V. Simplicity & Maintainability
Code should be as simple as possible. Avoid unnecessary complexity. Refactor regularly for clarity and maintainability.

## Additional Constraints
- Technology stack: React, TypeScript, Vite, CSS Modules.
- All API endpoints must be documented.
- No sensitive data stored in the front-end.

## Development Workflow
- All code changes require code review.
- Automated tests must pass before merging.
- Manual QA for major UI changes.

## Governance
This constitution supersedes all other practices. Amendments require documentation and approval by the project lead.

## Inclusiveness
The website must be designed to be inclusive and welcoming to all users, regardless of background or ability.
It must support people with indigenous names, transgender people, and people with disabilities. 
It must also be designed to be culturally sensitive and respectful.
